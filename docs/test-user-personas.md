## AI Triageの割り当てロジック（期待される動作）

### 優先順位1: スキルマッチング
チケットのカテゴリーに最も適したスキルを持つユーザーを選択

### 優先順位2: 負荷分散
同じスキルレベルの場合、現在の負荷が低いユーザーを優先

### 優先順位3: 緊急度
- **High/Urgent**: Aliceを優先（リーダーとして緊急対応）
- **Medium/Normal**: BobまたはCarolを優先
- **Low/Normal**: 最も負荷が低いユーザーを優先

## 使用方法

1. **Jiraでユーザーを作成**:
   - alice.johnson@example.com
   - bob.smith@example.com
   - carol.davis@example.com

2. **プロジェクトに「Agent」として追加**

3. **テストチケットを作成**して、AI Triageを実行

4. **期待される割り当て**と実際の割り当てを比較

5. **負荷状況を変更**して、異なるシナリオをテスト

---

## 注意事項

- 現在のAI Triageは**負荷が最も低いユーザーを優先**します（スキルマッチングは将来実装予定）
- Rovo Agent統合後は、より高度なスキルマッチングが可能になります
- テスト時は、各ユーザーに異なる数のチケットを割り当てて、負荷分散をテストしてください
