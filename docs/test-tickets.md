## Testing Tips

### 1. 担当者の負荷をテスト
各ユーザーに異なる数のチケットを割り当てて、負荷分散をテストしてください:
- Alice: 6件（高負荷）
- Bob: 4件（中負荷）
- Carol: 2件（低負荷）

### 2. スキルマッチングをテスト
各カテゴリーのチケットを作成して、適切な専門家に割り当てられるか確認:
- Network問題 → Alice
- Hardware問題 → Bob
- Account問題 → Carol

### 3. 優先度による割り当てをテスト
- High/Urgent → 専門家優先（Aliceの負荷が高い場合は他の専門家）
- Medium/Normal → 負荷が低いユーザー優先
- Low/Normal → 最も負荷が低いユーザー

### 4. エッジケースをテスト
- 空の説明文
- 非常に長い説明文
- 特殊文字を含む説明文
- 複数のカテゴリーに該当するチケット

### 5. 類似チケットをテスト
同じような内容のチケットを複数作成して、類似チケット検出をテスト

## Expected AI Behavior

### 分類の信頼度
- **High confidence (75-95%)**: 明確なキーワードが特定のカテゴリーに一致
- **Medium confidence (60-74%)**: いくつかのキーワードがあるが曖昧
- **Low confidence (0-59%)**: 明確なキーワードがない、または矛盾する情報

### 担当者の割り当てロジック（現在）
1. **負荷が最も低いユーザーを優先**（currentLoad順）
2. 将来的には、スキルマッチングも考慮される予定

### 担当者の割り当てロジック（将来）
1. **スキルマッチング**: チケットのカテゴリーに最適なスキルを持つユーザー
2. **負荷分散**: 同じスキルレベルの場合、負荷が低いユーザーを優先
3. **緊急度**: High/Urgentの場合、専門家を優先
