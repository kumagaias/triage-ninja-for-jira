# TriageNinja for Jira - プロダクト仕様書

## プロダクト概要

### アプリ名
**TriageNinja for Jira** 🥷

### タグライン
- "AI-Powered Intelligent Ticket Triage"
- "Master the art of AI triage"

### エレベーターピッチ
TriageNinjaは、AIを活用してJiraのサポートチケットを自動的に分類・担当者アサイン・解決策提案を行うForgeアプリです。手動トリアージの時間を80%削減し、サポートチームの生産性を劇的に向上させます。

## 解決する課題

### 課題1: 手動トリアージの時間コスト
**現状**:
- 1チケットあたり5-10分かかる
- 1日50件なら4-8時間（ほぼ1日分）
- 大量のチケットで担当者が疲弊

**影響**:
- 本来の業務（問題解決）に時間を使えない
- 顧客への初回応答が遅れる
- チームの士気低下

### 課題2: 担当者アサインのミスマッチ
**現状**:
- スキルに合わない担当者へのアサイン
- 専門知識がないため解決に時間がかかる
- 再アサインの手間

**影響**:
- 解決時間の長期化
- 顧客満足度の低下
- 担当者のフラストレーション

### 課題3: 負荷の不均等
**現状**:
- 特定の担当者に集中
- 新人には簡単なチケットのみ
- チーム全体の効率低下

**影響**:
- ベテランの燃え尽き
- 新人の成長機会損失
- チーム全体の生産性低下

### 課題4: 過去の知見が活用されない
**現状**:
- 類似チケットの解決策が見つからない
- 同じ問題を何度も調査
- ナレッジが属人化

**影響**:
- 解決時間の無駄
- 一貫性のない対応
- ナレッジの蓄積が進まない

## ソリューション

### コア機能

#### 1. 自動チケット分類 🤖
**機能**:
- AIがチケットの概要と説明を分析
- カテゴリー、サブカテゴリーを自動判定
- 優先度（High/Medium/Low）を判定
- 緊急度（Urgent/Normal）を評価
- 関連タグを自動付与

**ユーザー体験**:
1. 新規チケットが作成される
2. チケット詳細画面で「AI Triage」ボタンをクリック
3. 3秒以内にAIが分析結果を表示
4. 結果を確認して承認または修正
5. ワンクリックでチケットに反映

**価値**:
- 手動分類の時間を90%削減
- 一貫性のある分類
- 人的ミスの削減

#### 2. 最適な担当者アサイン 👤
**機能**:
- 過去の解決実績を分析
- スキルマッチング（カテゴリーごとの得意分野）
- 現在のワークロードを考慮
- 解決率・平均解決時間を基に最適化
- アサイン理由を明確に説明

**ユーザー体験**:
1. AI分析結果に推奨担当者が表示される
2. 担当者の名前、スキル、現在の負荷が見える
3. 選定理由が説明される（例: "VPN専門家、解決率95%"）
4. 代替候補も表示される
5. ワンクリックでアサイン完了

**価値**:
- スキルマッチングによる解決時間短縮
- 負荷分散による燃え尽き防止
- 新人への適切なチケット配分

#### 3. 類似チケット検索 🔍
**機能**:
- テキスト類似度分析
- 同じカテゴリーの過去チケットを検索
- 解決済みチケットの解決方法を表示
- 平均解決時間も表示
- 類似度スコアを表示

**ユーザー体験**:
1. AI分析結果に類似チケットが表示される
2. 最大5件の類似チケットが類似度順に並ぶ
3. 各チケットの解決方法がハイライト表示
4. チケットIDをクリックで詳細確認
5. 解決方法を参考に対応

**価値**:
- 過去の知見を即座に活用
- 解決時間の短縮
- 一貫性のある対応

#### 4. AI信頼度スコア ✨
**機能**:
- 分類の確実性をパーセンテージで表示
- 低信頼度（<70%）の場合は警告
- 信頼度に応じてUIの色を変更
- 信頼度の計算根拠を表示

**ユーザー体験**:
1. AI分析結果に信頼度スコアが表示される
2. 色分けで一目で判断（緑: 90%+、黄: 70-89%、赤: <70%）
3. 低信頼度の場合は手動確認を推奨
4. 計算根拠を確認できる

**価値**:
- AIの判断を適切に評価
- 低信頼度時の人間の介入
- 透明性のあるAI活用

#### 5. ダッシュボード 📊
**機能**:
- 統計カード（未トリアージ数、本日の処理数、時間削減率、AI精度）
- 未処理チケット一覧
- フィルタリング機能（優先度、カテゴリー）
- リアルタイム更新

**ユーザー体験**:
1. プロジェクトページでダッシュボードを開く
2. 一目でチーム全体の状況を把握
3. 未処理チケットを優先度順に確認
4. フィルターで特定のカテゴリーに絞り込み
5. クイックアクションでトリアージ実行

**価値**:
- チーム全体の可視化
- 優先順位の明確化
- マネージャーの意思決定支援

## ユーザーフロー

### メインフロー: チケットトリアージ

```
1. 新規チケット作成
   ↓
2. チケット詳細画面を開く
   ↓
3. 「AI Triage」ボタンをクリック
   ↓
4. AIが分析中（ローディング表示）
   ↓
5. 分析結果が表示される
   - カテゴリー: Network & Connectivity > VPN Issues
   - 優先度: High
   - 緊急度: Urgent
   - 信頼度: 94%
   - 推奨担当者: Alex Chen（VPN専門家、解決率95%）
   - 類似チケット: HELP-892（89%類似）
   ↓
6. 結果を確認
   - 適切なら「Approve」
   - 修正が必要なら手動調整
   ↓
7. ワンクリックでアサイン
   ↓
8. チケットが更新される
   - カテゴリー、優先度、担当者が設定
   - コメントに分析結果が記録
```

### サブフロー1: ダッシュボードから一括トリアージ

```
1. プロジェクトページでダッシュボードを開く
   ↓
2. 未トリアージチケット一覧を確認
   ↓
3. 優先度でフィルタリング（High優先）
   ↓
4. 各チケットの「Triage」ボタンをクリック
   ↓
5. AI分析結果を確認
   ↓
6. 一括承認または個別調整
   ↓
7. すべてのチケットがトリアージ完了
```

### サブフロー2: 類似チケットから解決策を参照

```
1. AI分析結果で類似チケットを確認
   ↓
2. 最も類似度の高いチケット（HELP-892）をクリック
   ↓
3. 過去のチケット詳細を確認
   - 問題: VPN接続タイムアウト
   - 解決方法: VPNクライアントのキャッシュをリセット
   - 解決時間: 20分
   ↓
4. 同じ解決方法を試す
   ↓
5. 問題が解決
   ↓
6. チケットをクローズ
```

## 用語集

### ビジネス用語

- **トリアージ (Triage)**: チケットを分類し、優先順位を付け、適切な担当者にアサインするプロセス
- **チケット (Ticket)**: サポートリクエストや問題報告を記録したJira Issue
- **アサイン (Assign)**: チケットを特定の担当者に割り当てること
- **解決率 (Resolution Rate)**: 担当者が解決したチケットの割合
- **ワークロード (Workload)**: 担当者が現在抱えているチケット数

### 技術用語

- **Forge**: Atlassianのサーバーレスアプリ開発プラットフォーム
- **Rovo Agent**: AtlassianのAIエージェント機能
- **Forge Storage**: Forgeアプリのデータ永続化ストレージ
- **Runs on Atlassian**: Atlassianインフラ上で動作する認証
- **JQL (Jira Query Language)**: Jiraのチケット検索言語

### AI用語

- **信頼度スコア (Confidence Score)**: AIの判断の確実性を示す0-100の数値
- **類似度 (Similarity)**: 2つのチケットがどれだけ似ているかを示す0-1の数値
- **プロンプトエンジニアリング**: AIに適切な指示を与えて精度を向上させる技術
- **意味的類似度 (Semantic Similarity)**: テキストの意味的な類似性

## 成功指標（KPI）

### プロダクトKPI

1. **採用率 (Adoption Rate)**
   - 目標: 80%のチームが週1回以上使用
   - 測定: アクティブユーザー数 / 総ユーザー数

2. **トリアージ自動化率**
   - 目標: 70%のチケットがAI承認
   - 測定: AI承認チケット数 / 総チケット数

3. **時間削減率**
   - 目標: 80%の時間削減
   - 測定: (手動時間 - AI時間) / 手動時間

4. **AI精度**
   - 目標: 95%の精度
   - 測定: 正しい分類数 / 総分類数

### ビジネスKPI

1. **顧客満足度 (CSAT)**
   - 目標: 20%向上
   - 測定: サポート満足度スコア

2. **初回応答時間 (First Response Time)**
   - 目標: 50%短縮
   - 測定: チケット作成から初回応答までの時間

3. **解決時間 (Resolution Time)**
   - 目標: 30%短縮
   - 測定: チケット作成から解決までの時間

4. **チケット処理数**
   - 目標: 2倍向上
   - 測定: 1日あたりの処理チケット数
